# Pre-Flight Check (사전 점검 스킬)

## 언제 사용하는가
모든 실행 작업(이메일 발송, 캘린더 등록, 파일 조작, 문서 작성, 웹 폼 제출) 시작 **직전**에 반드시 이 스킬을 실행한다.

## 실행 절차

### Step 1: LESSON 스캔
- MEMORY.md에서 `[LESSON]` 태그가 붙은 항목을 모두 읽는다
- 현재 작업과 카테고리가 같은 LESSON을 필터링한다
- 카테고리: 이메일, 캘린더, 문서작성, 데이터, 웹작업, 파일관리, 인코딩

### Step 2: 위험 패턴 매칭
현재 작업이 아래 조건에 해당하는지 체크한다:
- 과거에 실패한 것과 같은 유형의 작업인가?
- 한글이 포함된 작업인가? → korean-encoding 스킬 연동
- 첨부파일이 언급되었는데 실제로 첨부되었는가?
- 수신자가 올바른가? 외부 도메인이 포함되어 있는가?
- 되돌릴 수 없는 작업(삭제, 발송, 게시)인가?

### Step 3: 체크리스트 생성
관련 LESSON이 있으면 아래 형식으로 주인님께 보고한다:

⚡ **사전 점검 결과**:
✅ 관련 LESSON 1건 발견: [2026-03-03] 캘린더 한글 인코딩 → 유니코드 코드포인트 방식 필수
⚠️ 주의 사항: 한글 제목 포함 → korean-encoding 스킬 적용
📋 실행 계획: (작업 내용 요약)

**진행할까요?**

### Step 4: 승인 후 실행
- 주인님이 확인(ㅇㅇ, 응, 진행 등)하면 실행
- 주인님이 수정 요청하면 계획 수정 후 재확인

### Step 5: 실패 시 자동 기록
- 작업이 실패하거나 주인님이 수정을 요청하면:
    - 즉시 `[LESSON] YYYY-MM-DD | 카테고리 | 원인 | 올바른 방법` 형식으로 MEMORY.md에 추가
    - 같은 카테고리 LESSON이 3건 이상이면 "🔁 반복 주의 패턴"으로 승격

## 절대 규칙
- 이 스킬을 건너뛰지 않는다
- LESSON이 0건이어도 기본 체크(수신자, 첨부, 인코딩)는 수행한다
- 사전 점검 결과를 주인님에게 보여주지 않고 실행하지 않는다
