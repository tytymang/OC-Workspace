# Mail Consolidated Reporting

이 스킬은 흩어져 있는 메일과 첨부파일 정보를 취합하여 하나의 완성된 보고서로 만드는 고난도 업무 자동화 가이드입니다.

## 🛠 핵심 워크플로우

### 1. 다중 소스 데이터 수집
- **Outlook**: 메일 본문의 배경 맥락(누가, 언제, 왜 보냈는지)을 먼저 파악합니다.
- **Excel**: `pandas`를 이용해 시트별 데이터를 추출합니다. 한글 인코딩에 주의합니다.
- **PPT (텍스트)**: 슬라이드의 제목과 텍스트 상자 데이터를 우선 추출합니다.
- **PPT (이미지/OCR)**: 텍스트 추출로 내용이 비어 있는 경우, 슬라이드 내 '이미지' 존재 여부를 확인합니다.

### 2. 보안/DRM 우회 이미지 추출 (핵심 기법)
보안 솔루션(Fasoo 등)이 적용된 환경에서는 일반적인 `Export` 명령이 작동하지 않을 수 있습니다. 이때는 **클립보드 복사 방식**을 사용하여 이미지를 추출합니다.
- **방법**: PowerShell 또는 Python `win32com`을 이용해 `Shape.Copy()` 호출 -> `PIL.ImageGrab.grabclipboard()`로 이미지 저장.
- **연속 작업**: 대량의 슬라이드 작업 시 클립보드 경합을 방지하기 위해 각 작업 사이에 0.5~1초의 지연 시간을 둡니다.

### 3. 데이터 통합 및 정제
- **중복 제거**: 여러 파일에 걸쳐 중복된 담당자나 과제가 있을 경우, 가장 상세한 내용을 기준으로 통합합니다.
- **Human-Readable Text**: 보고서에는 HTML 태그나 기술적 기호를 제거하고, 불렛 포인트(•)와 간결한 문장을 사용합니다.
- **섹터/담당자 분리**: 보고 효율을 위해 소속 조직(섹터)과 담당자명을 별도 열로 분리하여 구성합니다.

## 📂 추천 도구 및 스크립트
- `python -m pip install pywin32 pillow pandas openpyxl`: 필수 라이브러리.
- 이미지 기반 OCR 시에는 OpenClaw의 `image` 도구를 사용하거나 Vision AI 모델에 이미지를 전달합니다.

## 📋 보고 원칙
- **결론 우선**: 통합 보고서의 핵심 요약을 최상단에 배치합니다.
- **파일 제공**: 표 형태의 보고뿐만 아니라, 주인님이 즉시 활용할 수 있는 통합 엑셀 파일(`.xlsx`)을 항상 함께 생성하여 제공합니다.
