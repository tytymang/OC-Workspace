# File Parser (파일 해석 스킬)

## 언제 사용하는가
- 주인님이 이미지(스크린샷, 사진)를 보내고 내용 분석을 요청할 때
- PDF, Excel, Word 문서의 내용을 읽거나 요약할 때
- 메일 첨부파일을 분석할 때
- 표, 차트, 도표가 포함된 파일을 해석할 때

## 파일 유형별 처리 절차

### 이미지 파일 (.png, .jpg, .bmp)
1. 이미지를 수신하면 먼저 전체 구조를 파악한다 (표인지, 텍스트인지, 차트인지)
2. **텍스트가 포함된 이미지**:
    - OCR이 필요하면 명시적으로 안내
    - 한글 텍스트는 특히 주의: 비슷한 글자 혼동 방지 (예: 회/횜, 의/읜)
    - 숫자와 날짜는 반드시 이중 확인
3. **표가 포함된 이미지**:
    - 행/열 구조를 먼저 파악
    - 셀 단위로 정확히 읽되, 불확실한 셀은 `[불명확]`으로 표시
    - 절대 추측으로 채우지 않는다
4. **해석 결과 보고 형식**:
    📄 **이미지 분석 결과**:
    **유형**: 표 / 텍스트 / 차트 / 혼합
    **내용**: (해석 결과)
    **확신도**: 높음 / 보통 / 낮음
    **불확실한 부분**: (있으면 명시)

### PDF 파일
1. 텍스트 추출 가능한 PDF → 직접 텍스트 추출
2. 스캔된 이미지 PDF → OCR 필요 안내
3. 한글 PDF는 인코딩 확인 (CID 폰트 등으로 깨질 수 있음)
4. 페이지 수가 많으면 먼저 목차/구조를 보고하고 주인님이 원하는 부분만 처리

### Excel 파일 (.xlsx, .csv)
1. 시트 목록과 각 시트의 행/열 수를 먼저 보고
2. 한글 헤더가 있으면 정확히 읽었는지 확인
3. 수식이 포함된 셀은 수식과 결과값을 구분하여 보고
4. CSV의 경우 인코딩 자동 감지 (UTF-8, EUC-KR, CP949)

### Word 파일 (.docx)
1. 본문 텍스트 + 표 + 이미지 순서로 구조 파악
2. 추적 변경(Track Changes)이 있으면 별도 표시
3. 머리글/바닥글 내용도 포함

## 공통 규칙

### 정확성 원칙
- 읽을 수 없는 부분은 **"확인 불가"**로 명시한다 — 절대 추측하지 않는다
- 숫자(금액, 날짜, 수량)는 원본과 반드시 대조한다
- 사람 이름이 포함되어 있으면 마스킹 규칙 적용 (개인정보 보호)

### 한글 특화 처리
- 한글 OCR 결과는 자주 틀리는 글자 쌍을 체크:
    - 회↔횜, 의↔읜, 원↔웬, 부↔뷰
- 불확실한 한글은 앞뒤 문맥으로 추론하되, 추론임을 명시

### 대용량 파일
- 100페이지 이상 PDF, 10,000행 이상 Excel은 먼저 구조만 보고
- 주인님에게 "어떤 부분을 분석할까요?" 확인 후 진행
- 전체 처리가 필요하면 소요 시간 예상치를 먼저 안내

### 개인정보 보호 연동
- 파일 내 개인정보(이름, 전화번호, 이메일 등) 발견 시:
    - 요약에서 마스킹 처리
    - 메모리에는 기록하지 않음
    - 주인님에게 "개인정보가 포함되어 있습니다" 알림
